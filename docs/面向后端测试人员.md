# TalkABC 后端服务 - 后端测试人员入门指南

## 一、项目概述

基于 Go + Gin + GORM + PostgreSQL + Redis 开发的社交应用后端 API 服务，支持 WebSocket 实时通信。

### 1.1 技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| Gin | HTTP Web框架 | 高性能路由和中间件 |
| GORM | ORM数据库操作 | 简化数据库操作 |
| PostgreSQL | 关系型数据库 | 存储用户、消息、动态等数据 |
| Redis | 缓存/实时数据存储 | 存储验证码、在线状态、会话管理 |
| JWT | 用户认证 | 无状态令牌认证 |
| zap | 结构化日志 | 高性能日志库 |

## 四、创建测试文件和测试方法

### 4.1 测试文件命名规范

| 测试类型 | 命名格式 | 示例 | 是否需要认证 |
|----------|----------|------|-------------|
| 集成测试（无需token） | `ff_xxx_test.go` | `ff_register_test.go` | 否 |
| 集成测试（需token） | `fftoken_xxx_test.go` | `fftoken_collect_profile_test.go` | 是 |
| 单元测试 | `unit_xxx_test.go` | `unit_auth_register_test.go` | 视情况 |

### 4.2 创建测试文件的步骤

#### 步骤1：确定测试类型

根据要测试的功能确定测试类型：

```
判断逻辑：
- 如果是单个函数/方法的测试 → 单元测试（unit_xxx_test.go）
- 如果是完整业务流程测试 → 集成测试
  - 流程中不需要认证 → ff_xxx_test.go
  - 流程中需要认证 → fftoken_xxx_test.go
```

#### 步骤2：创建测试文件

在 `tests/` 目录下创建测试文件：

```bash
# 示例：创建登录流程集成测试文件
touch tests/ff_login_test.go

# 示例：创建资料收集流程集成测试文件（需要认证）
touch tests/fftoken_collect_profile_test.go

# 示例：创建注册接口单元测试文件
touch tests/unit_auth_register_test.go
```

#### 步骤3：编写测试文件基本结构

### 4.3 单元测试编写指南

#### 4.3.1 单元测试模板

```go
package test

import (
    "backend/internal/handler"
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"

    "github.com/gin-gonic/gin"
)

func init() {
    InitTest()
}

// ==================== xxx接口单元测试 ====================

func TestXxx_InvalidParams(t *testing.T) {
    router := gin.New()
    router.POST("/v1/xxx", handler.XxxHandler)

    tests := []struct {
        name string
        body string
    }{
        {"Empty body", "{}"},
        {"Missing required field", `{"other": "value"}`},
        {"Invalid JSON", "invalid json"},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            req, _ := http.NewRequest("POST", "/v1/xxx", strings.NewReader(tt.body))
            req.Header.Set("Content-Type", "application/json")
            resp := httptest.NewRecorder()

            router.ServeHTTP(resp, req)

            if resp.Code != http.StatusBadRequest {
                t.Errorf("Expected status %d, got %d", http.StatusBadRequest, resp.Code)
            }
        })
    }
}
```

#### 4.3.2 单元测试要点

| 要点 | 说明 |
|------|------|
| 测试范围 | 测试单个接口/函数，不依赖其他接口 |
| 请求方式 | 使用 `application/json` 或 `application/x-www-form-urlencoded` |
| 验证目标 | 验证参数校验、错误处理、响应格式 |
| 数据库依赖 | 一般不需要数据库，如需则使用测试数据库 |

```go
package test

import (
    "backend/internal/config"
    "backend/internal/handler"
    "encoding/json"
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"

    "github.com/gin-gonic/gin"
)

func init() {
    InitTest()
}

// TestXxx_FullFlow xxx完整流程集成测试
// 测试流程：
//   Step1: 准备阶段（清理环境）
//   Step2: 第一步操作（如发送验证码）
//   Step3: 第二步操作（如注册/登录）
//   Step4: 验证结果（如数据库记录、Redis状态）
func TestXxx_FullFlow(t *testing.T) {
    // 检查依赖是否初始化
    if config.DB == nil || mockSMSGateway == nil {
        t.Skip("Database or Mock SMS gateway not initialized, skipping test")
    }

    // 初始化路由
    router := gin.New()
    router.GET("/v1/code/sms", handler.SendSMSCode)
    router.POST("/v1/xxx", handler.XxxHandler)

    // 测试数据
    phoneNum := "13900139003"

    // 清理测试环境（重要：防止测试污染）
    config.RDB.FlushDB(config.RDB.Context())
    mockSMSGateway.ClearSentMessages()

    // ==================== Step1: 发送验证码 ====================
    t.Run("Step1_SendSMSCode", func(t *testing.T) {
        req, _ := http.NewRequest("GET", "/v1/code/sms?phonenum="+phoneNum+"&tag=xxx", nil)
        resp := httptest.NewRecorder()
        router.ServeHTTP(resp, req)

        if resp.Code != http.StatusOK {
            t.Fatalf("SendSMSCode failed: %s", resp.Body.String())
        }

        // 验证Mock短信网关收到请求
        sentMsgs := mockSMSGateway.GetSentMessages()
        if len(sentMsgs) != 1 {
            t.Fatalf("Expected 1 sent message, got %d", len(sentMsgs))
        }
    })

    // ==================== Step2: 执行xxx操作 ====================
    t.Run("Step2_XxxOperation", func(t *testing.T) {
        sentMsgs := mockSMSGateway.GetSentMessages()
        if len(sentMsgs) == 0 {
            t.Skip("No SMS code sent in previous step")
        }
        code := sentMsgs[0].Code

        // 构造请求
        formData := strings.NewReader("phonenum=" + phoneNum + "&code=" + code)
        req, _ := http.NewRequest("POST", "/v1/xxx", formData)
        req.Header.Set("Content-Type", "application/x-www-form-urlencoded")
        resp := httptest.NewRecorder()

        router.ServeHTTP(resp, req)

        // 验证响应
        if resp.Code != http.StatusOK {
            t.Fatalf("XxxOperation failed: %s", resp.Body.String())
        }

        // 解析响应
        var result map[string]interface{}
        if err := json.Unmarshal(resp.Body.Bytes(), &result); err != nil {
            t.Fatalf("Failed to unmarshal response: %v", err)
        }

        if result["code"] != float64(0) {
            t.Errorf("Expected code 0, got %v", result["code"])
        }
    })

    // ==================== Step3: 验证结果 ====================
    t.Run("Step3_VerifyResult", func(t *testing.T) {
        // 验证数据库记录
        // 验证Redis状态
        // 验证其他副作用
    })
}
```

#### 4.4.2 集成测试模板（需要token）

```go
package test

import (
    "backend/internal/config"
    "backend/internal/handler"
    "encoding/json"
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"

    "github.com/gin-gonic/gin"
)

func init() {
    InitTest()
}

// TestXxx_FullFlow xxx完整流程集成测试（需要认证）
func TestXxx_FullFlow(t *testing.T) {
    if config.DB == nil || mockSMSGateway == nil {
        t.Skip("Database or Mock SMS gateway not initialized, skipping test")
    }

    // 初始化路由（包含需要认证的接口）
    router := gin.New()
    router.GET("/v1/code/sms", handler.SendSMSCode)
    router.POST("/v1/register", handler.Register)
    router.POST("/v1/login/code", handler.LoginByCode)
    router.POST("/v1/xxx", handler.XxxHandler) // 需要认证

    // 测试数据
    phoneNum := "13900139004"
    password := "Password123"

    // 清理测试环境
    config.RDB.FlushDB(config.RDB.Context())
    mockSMSGateway.ClearSentMessages()

    // ==================== Step1: 注册用户 ====================
    t.Run("Step1_Register", func(t *testing.T) {
        // 发送验证码
        req, _ := http.NewRequest("GET", "/v1/code/sms?phonenum="+phoneNum+"&tag=register", nil)
        resp := httptest.NewRecorder()
        router.ServeHTTP(resp, req)

        // 获取验证码并注册
        sentMsgs := mockSMSGateway.GetSentMessages()
        code := sentMsgs[0].Code

        formData := strings.NewReader("phonenum=" + phoneNum + "&code=" + code + "&password=" + password)
        registerReq, _ := http.NewRequest("POST", "/v1/register", formData)
        registerReq.Header.Set("Content-Type", "application/x-www-form-urlencoded")
        registerResp := httptest.NewRecorder()
        router.ServeHTTP(registerResp, registerReq)
    })

    // ==================== Step2: 登录获取token ====================
    var accessToken string
    t.Run("Step2_Login", func(t *testing.T) {
        // 发送登录验证码
        req, _ := http.NewRequest("GET", "/v1/code/sms?phonenum="+phoneNum+"&tag=login", nil)
        resp := httptest.NewRecorder()
        router.ServeHTTP(resp, req)

        sentMsgs := mockSMSGateway.GetSentMessages()
        code := sentMsgs[len(sentMsgs)-1].Code // 获取最新发送的验证码

        // 验证码登录
        formData := strings.NewReader("phonenum=" + phoneNum + "&code=" + code)
        loginReq, _ := http.NewRequest("POST", "/v1/login/code", formData)
        loginReq.Header.Set("Content-Type", "application/x-www-form-urlencoded")
        loginResp := httptest.NewRecorder()
        router.ServeHTTP(loginResp, loginReq)

        // 解析token
        var result map[string]interface{}
        json.Unmarshal(loginResp.Body.Bytes(), &result)
        data := result["data"].(map[string]interface{})
        accessToken = data["access_token"].(string)
    })

    // ==================== Step3: 执行需要认证的操作 ====================
    t.Run("Step3_XxxOperation", func(t *testing.T) {
        req, _ := http.NewRequest("POST", "/v1/xxx", strings.NewReader(`{"param": "value"}`))
        req.Header.Set("Content-Type", "application/json")
        req.Header.Set("Authorization", "Bearer "+accessToken) // 添加认证token
        resp := httptest.NewRecorder()

        router.ServeHTTP(resp, req)

        if resp.Code != http.StatusOK {
            t.Fatalf("XxxOperation failed: %s", resp.Body.String())
        }
    })
}
```

### 4.5 测试方法编写注意事项

#### 4.5.1 环境准备注意事项

| 注意事项 | 说明 |
|----------|------|
| 清理环境 | 每次测试前清理Redis和Mock数据，防止测试污染 |
| 检查依赖 | 使用 `t.Skip()` 跳过依赖未初始化的测试 |
| 初始化路由 | 使用 `gin.New()` 创建独立的测试路由，避免相互影响 |

#### 4.5.2 请求构造注意事项

| 注意事项 | 说明 |
|----------|------|
| 请求方式 | GET请求用 `http.NewRequest("GET", ...)`，POST请求用 `http.NewRequest("POST", ...)` |
| 请求头 | 根据接口要求设置 `Content-Type`，如 `application/json` 或 `application/x-www-form-urlencoded` |
| 认证头 | 需要认证的接口添加 `Authorization: Bearer <token>` |
| 参数传递 | GET请求参数放在URL中，POST请求参数放在body中 |

#### 4.5.3 响应验证注意事项

| 注意事项 | 说明 |
|----------|------|
| 状态码验证 | 验证HTTP状态码是否符合预期 |
| 响应格式验证 | 验证响应是否符合 `{code, msg, data}` 格式 |
| 业务码验证 | 验证 `code` 字段是否为0（成功）或预期错误码 |

## 六、统一响应格式

```json
{
  "code": 0,           // 状态码：0表示成功，其他值表示失败
```

## 七、认证方式

### 7.1 JWT Token认证

所有需要登录的接口，请求头中必须携带 `Authorization`：

```bash
curl -X GET "http://localhost:8080/api/v1/users" \
  -H "Authorization: Bearer <token>"
```

### 7.2 双令牌机制

系统采用双令牌机制提升安全性：

| 令牌类型 | 有效期 | 用途 | 存储位置 |
|----------|--------|------|----------|
| access_token | 2小时 | 日常API访问认证 | 请求头Authorization |
| refresh_token | 7天 | 获取新的access_token | 客户端本地存储 |

### 7.3 登录流程

```
Step 1: 获取验证码
GET /api/v1/auth/code-sms?phonenum=13800138000

Step 2: 用户登录（验证码）
POST /api/v1/auth/login/code
Body: { phonenum, code, device_id }

Step 3: 用户登录（密码）
POST /api/v1/auth/login/password
Body: { phonenum, password }

Step 4: 获取令牌（返回 access_token 和 refresh_token）
```

## 十五、常见问题

### Q1: 测试失败如何排查？
A: 查看测试输出，检查数据库连接、Redis连接、配置文件是否正确

### Q2: 如何查看测试覆盖率？
A: `go test ./tests/... -coverprofile=coverage.out && go tool cover -html=coverage.out`

### Q3: 如何模拟短信验证码？
A: 测试框架已使用Mock短信网关，可以通过 `mockSMSGateway.GetLastCode(phoneNum)` 获取发送的验证码

### Q4: 测试环境数据库被占用？
A: 确保测试使用独立的数据库（talkabc_test）和Redis DB（1）
