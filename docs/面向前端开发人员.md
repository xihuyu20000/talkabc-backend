# TalkABC 后端服务 - 前端开发人员入门指南

## 一、快速开始

### 1.1 启动后端服务

```bash
# 进入项目目录
cd talkabc-backend

# 设置开发环境
export APP_ENV=dev

# 启动服务
go run ./cmd/server

# 或使用构建脚本
build_dev.bat
```

服务启动后：
- API地址：`http://localhost:8080/api/v1`
- Swagger文档：`http://localhost:8080/swagger/index.html`

### 1.2 开发环境配置

前端开发时，请确保后端配置为开发模式：

```yaml
# config/config.dev.yaml
cors:
  origins:
    - "*"  # 开发环境允许所有来源
```

## 二、认证方式

### 2.1 JWT Token认证

所有需要登录的接口，请求头中必须携带 `Authorization`：

```dart
import 'package:dio/dio.dart';

final dio = Dio();

// 请求示例
Response response = await dio.get(
  '/api/v1/users',
  options: Options(
    headers: {
      'Authorization': 'Bearer $token',
      'Content-Type': 'application/json',
    },
  ),
);
```

### 2.2 登录流程

```dart
// Step 1: 获取验证码（tag为必填项，允许的值：register、login、change_phone、reset_password）
Response smsResponse = await dio.get(
  '/api/v1/auth/code-sms',
  queryParameters: {'phonenum': '13800138000', 'tag': 'register'},
);

// Step 2: 用户登录（验证码登录）
Response loginResponse = await dio.post(
  '/api/v1/auth/login/code',
  data: {
    'phonenum': '13800138000',
    'code': '123456',
    'device_id': 'device_123',
  },
);

// Step 3: 保存令牌（返回 access_token 和 refresh_token）
final data = loginResponse.data['data'];
final accessToken = data['access_token'];
final refreshToken = data['refresh_token'];

// 使用flutter_secure_storage保存到安全存储
final storage = FlutterSecureStorage();
await storage.write(key: 'access_token', value: accessToken);
await storage.write(key: 'refresh_token', value: refreshToken);
```

### 2.3 双令牌机制

系统采用双令牌机制提升安全性：

| 令牌类型 | 有效期 | 用途 | 存储位置 |
|----------|--------|------|----------|
| access_token | 2小时 | 日常API访问认证 | 请求头Authorization |
| refresh_token | 7天 | 获取新的access_token | FlutterSecureStorage |

### 2.4 令牌刷新

当 access_token 过期时，使用 refresh_token 获取新令牌：

```dart
import 'package:dio/dio.dart';
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

final storage = FlutterSecureStorage();
final dio = Dio();

// 刷新令牌
Future<void> refreshToken() async {
  final refreshToken = await storage.read(key: 'refresh_token');
  
  Response response = await dio.post(
    '/api/v1/auth/refresh-token',
    data: {'refresh_token': refreshToken},
  );
  
  if (response.data['code'] == 0) {
    final data = response.data['data'];
    await storage.write(key: 'access_token', value: data['access_token']);
    await storage.write(key: 'refresh_token', value: data['refresh_token']);
  }
}

// 配置Dio拦截器自动刷新token
void setupDio() {
  dio.interceptors.add(InterceptorsWrapper(
    onError: (DioException error, ErrorInterceptorHandler handler) async {
      if (error.response?.statusCode == 401) {
        await refreshToken();
        // 重新发起请求
        final newToken = await storage.read(key: 'access_token');
        final options = error.requestOptions;
        options.headers['Authorization'] = 'Bearer $newToken';
        final response = await dio.fetch(options);
        handler.resolve(response);
        return;
      }
      handler.reject(error);
    },
  ));
}
```

**安全规则：**
- refresh_token存储在Redis中，退出登录时立即失效
- 每次刷新生成新的refresh_token（令牌轮转）
- JWT中间件验证请求中的token与Redis中存储的token一致，防止旧token滥用

### 2.5 用户Uid说明

- 用户对外唯一标识使用雪花ID（字符串类型，varchar(20)）
- 数据库自增ID仅内部使用，不对外暴露
- 防止爬虫遍历用户数据

## 三、统一响应格式

### 3.1 成功响应

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "user": { ... }
  }
}
```

### 3.2 错误响应

```json
{
  "code": 1,
  "msg": "错误描述信息",
  "data": null
}
```

### 3.3 通用响应模型

```dart
import 'package:json_annotation/json_annotation.dart';

part 'api_response.g.dart';

@JsonSerializable()
class ApiResponse<T> {
  @JsonKey(name: 'code')
  final int code;

  @JsonKey(name: 'msg')
  final String msg;

  @JsonKey(name: 'data')
  final T? data;

  ApiResponse({
    required this.code,
    required this.msg,
    this.data,
  });

  factory ApiResponse.fromJson(Map<String, dynamic> json) =>
      _$ApiResponseFromJson(json);

  Map<String, dynamic> toJson() => _$ApiResponseToJson(this);

  bool get isSuccess => code == 0;
}
```

### 3.4 常见错误码

| 错误码 | 含义 |
|--------|------|
| 0 | 成功 |
| 1 | 通用错误 |
| 400 | 请求参数错误 |
| 401 | 未授权（Token无效或过期） |
| 403 | 禁止访问 |
| 500 | 服务器内部错误 |

## 五、WebSocket实时通信

### 5.1 连接方式

客户端通过 WebSocket 协议连接服务器，需在 URL 参数中携带 JWT token：

```dart
import 'package:web_socket_channel/web_socket_channel.dart';

final channel = WebSocketChannel.connect(
  Uri.parse('ws://localhost:8080/ws?token=your_jwt_token&deviceId=your_device_id'),
);
```

### 5.2 连接事件处理

```dart
import 'package:web_socket_channel/web_socket_channel.dart';
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

final storage = FlutterSecureStorage();

Future<void> connectWebSocket() async {
  final accessToken = await storage.read(key: 'access_token');
  final deviceId = 'your_device_id';
  
  final channel = WebSocketChannel.connect(
    Uri.parse('ws://localhost:8080/ws?token=$accessToken&deviceId=$deviceId'),
  );

  // 监听消息
  channel.stream.listen(
    (message) {
      final Map<String, dynamic> data = Map.from(json.decode(message));
      handleMessage(data);
    },
    onError: (error) {
      print('WebSocket错误: $error');
    },
    onDone: () {
      print('WebSocket连接关闭');
    },
  );
}
```

### 5.3 消息协议

服务端推送的消息采用 JSON 格式，包含以下字段：

```json
{
  "type": "chat",      // 消息类型
  "from_uid": "1",     // 发送者用户ID（字符串类型）
  "to_uid": "2",       // 接收者用户ID（字符串类型）
  "data": {}           // 消息数据
}
```

### 5.4 服务端推送消息类型

| 类型 | 说明 | 数据结构 |
|------|------|----------|
| `chat` | 聊天消息 | `{"id": 1, "text": "消息内容", "file_url": "图片/音视频URL", "msg_type": 1, "send_time": 1678901234}` |
| `withdraw` | 撤回消息 | `{"msg_id": 123}` |
| `friend_request` | 好友申请 | `{"from_uid": "1", "from_name": "用户名", "from_avatar": "头像URL"}` |
| `comment` | 评论通知 | `{"moment_id": 1, "from_uid": "2", "from_name": "用户名", "text": "评论内容"}` |

### 5.7 消息类型说明

**msg_type（聊天消息类型）：**
- 1 - 文本消息
- 2 - 图片消息
- 3 - 语音消息
- 4 - 视频消息
- 5 - 文件消息

**online（在线状态）：**
- 0 - 离线
- 1 - 在线

### 5.8 在线状态管理

系统通过 Redis 存储和管理用户在线状态：

**心跳机制：**
- 客户端每 30s 发送 ping 消息
- 服务端收到 ping 后更新 Redis Key 过期时间为 90s
- 超过 90s 未收到 ping，Key 自动过期，判定用户离线

**多端登录：**
- 支持手机、平板、网页同时在线
- 每个设备有独立的 deviceId
- 任意设备发送 ping 会续期整组在线状态

### 6.2 检查资料状态

```dart
import 'package:dio/dio.dart';
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

final dio = Dio();
final storage = FlutterSecureStorage();

Future<bool> checkProfileStatus() async {
  final accessToken = await storage.read(key: 'access_token');
  
  Response response = await dio.get(
    '/api/v1/profile/status',
    options: Options(
      headers: {'Authorization': 'Bearer $accessToken'},
    ),
  );
  
  if (response.data['code'] == 0) {
    return response.data['data']['profile_completed'] == 1;
  }
  return false;
}
```

## 七、安全规则注意事项

### 7.1 昵称安全规则

昵称需通过以下安全校验：
1. **长度限制**：2-20个字符
2. **字符类型**：仅允许中文、英文、数字、下划线、连字符
3. **敏感词过滤**：禁止包含暴力、色情、政治敏感、诈骗等违规词汇
4. **URL过滤**：禁止包含 http/https/www 等超链接
5. **HTML过滤**：禁止包含 HTML 标签
6. **XSS过滤**：禁止包含 XSS 攻击代码

### 7.2 个性签名安全规则

个性签名需通过以下安全校验：
1. **长度限制**：最大200字符
2. **敏感词过滤**：禁止包含暴力、色情、政治敏感、诈骗等违规词汇
3. **URL过滤**：禁止包含 http/https/www 等超链接
4. **HTML过滤**：禁止包含 HTML 标签
5. **JavaScript过滤**：禁止包含脚本代码（如 `<script>`、`eval()`、`alert()`）
6. **SQL注入过滤**：禁止包含 SQL 注入代码（如 `SELECT`、`INSERT`、`OR 1=1`）
7. **XSS过滤**：禁止包含 XSS 攻击代码（如 `onclick=`、`onerror=`）

## 九、常见问题

### Q1: 为什么登录后调用接口返回401？
A: 检查 `Authorization` 头是否正确格式：`Bearer token`（注意空格）

### Q2: 跨域问题如何解决？
A: 开发环境已配置 `cors: origins: ["*"]`，生产环境需配置具体域名。Flutter开发时可通过Dio配置baseUrl解决。

### Q3: 图片上传后如何访问？
A: 上传文件存放在 `./uploads/` 目录，访问地址：`http://localhost:8080/uploads/avatars/xxx.jpg`

### Q4: WebSocket连接失败？
A: 检查Token是否有效，确保URL格式正确：`ws://host/ws?token=xxx&deviceId=xxx`

### Q5: 如何处理Token过期？
A: 配置Dio拦截器，捕获401错误，使用refresh_token调用 `/api/v1/auth/refresh-token` 获取新令牌，然后重试请求

### Q6: 如何安全存储Token？
A: 使用 `flutter_secure_storage` 库，不要使用 `shared_preferences` 存储敏感信息

### Q7: 如何生成deviceId？
A: 可使用 `device_info_plus` 库获取设备唯一标识，或生成随机UUID作为设备ID
