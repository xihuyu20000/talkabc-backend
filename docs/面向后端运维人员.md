# TalkABC 后端服务 - 后端运维人员入门指南

## 一、项目概述

基于 Go + Gin + GORM + PostgreSQL + Redis 开发的社交应用后端 API 服务，支持 WebSocket 实时通信。

### 1.1 技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| Gin | HTTP Web框架 | 高性能路由和中间件 |
| GORM | ORM数据库操作 | 简化数据库操作 |
| PostgreSQL | 关系型数据库 | 存储用户、消息、动态等数据 |
| Redis | 缓存/实时数据存储 | 存储验证码、在线状态、会话管理 |
| JWT | 用户认证 | 无状态令牌认证 |
| zap | 结构化日志 | 高性能日志库 |

| 环境 | CPU | 内存 | 硬盘 | 带宽 |

|------|------|------|

| Nginx | >= 1.18 | 反向代理 |
| Certbot | >= 2.0 | SSL证书 |

### 2.3 Rocky Linux 10 简介

Rocky Linux 是一个开源、社区驱动的企业级 Linux 发行版，兼容 RHEL/CentOS。

**选择理由：**
- ✅ 长期支持（LTS），生命周期长达10年
- ✅ 完全兼容 RHEL/CentOS
- ✅ 社区活跃，更新及时
- ✅ 稳定可靠，适合生产环境
- ✅ 安全加固，默认启用 SELinux

**最小化安装要求：**
- 内存：至少 2GB
- 硬盘：至少 20GB
- 网络：至少 1Mbps

## 三、配置说明

### 3.1 多环境配置

项目支持多环境配置，配置文件位于 `config/` 目录下：

| 文件 | 说明 | 是否上传Git |
|------|------|-------------|
| `config/config.yaml` | 默认配置（公共部分） | 是 |
| `config/config.dev.yaml` | 开发环境配置 | 否 |
| `config/config.test.yaml` | 测试环境配置 | 否 |
| `config/config.prod.yaml` | 生产环境配置（含环境变量引用） | 否 |

### 3.2 配置文件加载优先级

1. **APP_CONFIG 环境变量**：指定完整配置文件路径
2. **APP_ENV 环境变量**：根据环境选择配置文件（`config/config.{APP_ENV}.yaml`）
3. **默认配置文件**：`config/config.yaml`
4. **兼容旧版本**：`./config.yaml`

### 3.3 环境变量配置

项目支持通过环境变量覆盖配置文件中的设置，以下是常用环境变量：

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| APP_ENV | 运行环境（dev/test/prod） | dev |
| APP_CONFIG | 配置文件路径 | config/config.yaml |
| DB_HOST | 数据库主机 | localhost |
| DB_PORT | 数据库端口 | 5432 |
| DB_USER | 数据库用户名 | postgres |
| DB_PASSWORD | 数据库密码 | 空 |
| DB_NAME | 数据库名称 | talkabc |
| REDIS_HOST | Redis主机 | localhost |
| REDIS_PORT | Redis端口 | 6379 |
| REDIS_PASSWORD | Redis密码 | 空 |
| REDIS_DB | Redis数据库编号 | 0 |
| JWT_SECRET | JWT签名密钥 | 空 |
| SMS_ACCESS_KEY | 短信服务AccessKey | 空 |
| SMS_SECRET_KEY | 短信服务SecretKey | 空 |

### 3.4 Apifox API管理

项目使用 **Apifox** 作为API管理平台，运维人员可以通过Apifox进行接口监控和故障排查。

**Apifox配置文件：** `talkabc.apifox.json`

**导入步骤：**
1. 下载并安装Apifox：[https://www.apifox.com/download](https://www.apifox.com/download)
2. 打开Apifox客户端，点击"导入/导出" → "导入数据" → "从文件导入"
3. 选择项目根目录下的 `talkabc.apifox.json` 文件
4. 点击"确定"完成导入

**各环境配置：**

| 环境 | 基础URL | 说明 |
|------|---------|------|
| 开发环境 | http://localhost:8080 | 本地开发测试 |
| 测试环境 | http://test-api.talkabc.com | 测试服务器 |
| 生产环境 | http://api.talkabc.com | 生产服务器 |

**运维人员常用功能：**

- 📊 **接口监控**：监控接口可用性和响应时间
- 🔍 **故障排查**：查看接口请求日志，定位问题
- 📋 **API文档**：查阅接口定义，了解接口功能
- 🧪 **健康检查**：定期测试关键接口，确保服务正常

**配置接口监控：**
1. 在Apifox中选择要监控的接口
2. 点击"监控" → "新建监控任务"
3. 设置监控频率和告警条件
4. 配置告警通知方式（邮件、钉钉、企微等）

**告警配置：**
- 响应时间超过500ms：警告
- 响应时间超过1000ms：严重
- 接口返回非200状态码：错误
- 接口连续3次请求失败：紧急

### 3.5 各环境配置差异

| 配置项 | 开发环境 | 测试环境 | 生产环境 |
|--------|----------|----------|----------|
| 日志级别 | debug | info | warn |
| 日志格式 | console | json | json |
| 日志输出 | console | both | file |
| CORS | `*` | 指定域名 | 指定域名 |
| 数据库 | talkabc | talkabc_test | 环境变量 |
| Redis DB | 0 | 1 | 0 |
| SSL | 否 | 否 | 是 |

### 3.5 配置文件示例

```yaml
# 系统配置
system:
  reset: 0                 # 重置标志(0=不重置, 1=重置所有数据)

# 日志配置
logger:
  level: warn           # 日志级别: debug, info, warn, error, fatal
  format: json          # 日志格式: console, json
  output: file          # 输出方式: console, file, both
  file_path: ./logs/app.log
  max_size: 100         # 单个日志文件最大大小(MB)
  max_backups: 30       # 保留的最大日志文件数
  max_age: 7            # 日志文件保留天数
  compress: true        # 是否压缩归档日志

# 安全配置
security:
  sms_valid_minutes: 5              # 短信验证码有效期（分钟）
  sms_cooldown_seconds: 60          # 短信发送冷却时间（秒）
  sms_hourly_limit: 10              # 每小时发送次数限制
  ip_register_hourly_limit: 10      # 注册时每小时每个IP发送次数限制
  ip_login_minute_limit: 10         # 登录时每分钟每个IP登录次数限制
  login_failure_lock_minutes: 5     # 登录失败锁定时间（分钟）

# 服务器配置
server:
  port: 8080              # 服务监听端口

# 数据库配置
database:
  host: "${DB_HOST}"         # 数据库服务器地址
  port: 5432             # PostgreSQL默认端口
  user: "${DB_USER}"         # 数据库用户名
  password: "${DB_PASSWORD}" # 数据库密码
  dbname: "${DB_NAME}"       # 数据库名称
  sslmode: require        # 生产环境启用SSL

# JWT认证配置
jwt:
  secret: "${JWT_SECRET}" # JWT签名密钥
  expires_hour: 24        # Token过期时间（小时）

# 文件上传配置
upload:
  avatar_path: ./uploads/avatars    # 头像上传目录
  moment_path: ./uploads/moments     # 动态上传目录
  message_path: ./uploads/messages   # 消息文件上传目录

# Redis配置
redis:
  host: "${REDIS_HOST}"              # Redis服务器地址
  port: 6379                      # Redis默认端口
  password: "${REDIS_PASSWORD}"    # Redis密码
  db: 0                           # Redis数据库编号

# CORS配置
cors:
  origins:
    - "https://talkabc.com"
    - "https://www.talkabc.com"
  methods:
    - GET
    - POST
    - PUT
    - DELETE
    - OPTIONS
  headers:
    - Origin
    - Content-Type
    - Authorization
    - Accept
    - X-Requested-With
  credentials: true
```

## 四、部署流程

### 4.1 Rocky Linux 10 初始化配置

```bash
# Step 1: 更新系统
sudo dnf update -y

# Step 2: 安装 EPEL 源（用于安装额外软件）
sudo dnf install -y epel-release

# Step 3: 安装基础工具
sudo dnf install -y wget curl git vim net-tools

# Step 4: 安装防火墙
sudo dnf install -y firewalld
sudo systemctl enable firewalld
sudo systemctl start firewalld

# Step 5: 安装 SELinux 工具
sudo dnf install -y setools-console
```

### 4.2 安装依赖服务

```bash
# Step 1: 安装 PostgreSQL 14
sudo dnf install -y postgresql-server postgresql-contrib
sudo postgresql-setup --initdb
sudo systemctl enable postgresql
sudo systemctl start postgresql

# Step 2: 安装 Redis
sudo dnf install -y redis
sudo systemctl enable redis
sudo systemctl start redis

# Step 3: 安装 Nginx
sudo dnf install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx

# Step 4: 安装 Go（可选，用于编译）
sudo dnf install -y golang
```

### 4.3 配置数据库

```bash
# Step 1: 创建数据库用户和数据库
sudo -u postgres psql -c "CREATE USER talkabc WITH PASSWORD 'your_secure_password';"
sudo -u postgres createdb -O talkabc talkabc

# Step 2: 配置 PostgreSQL 允许本地连接
sudo sed -i "s/ident/md5/g" /var/lib/pgsql/data/pg_hba.conf
sudo systemctl restart postgresql

# Step 3: 验证数据库连接
psql -h localhost -U talkabc -d talkabc -c "SELECT 1;"
```

### 4.4 配置 Redis

```bash
# Step 1: 设置 Redis 密码
sudo sed -i "/^# requirepass/s/^# //" /etc/redis.conf
sudo sed -i "s/requirepass foobared/requirepass your_redis_password/" /etc/redis.conf

# Step 2: 绑定本地地址（生产环境不要绑定0.0.0.0）
sudo sed -i "s/bind 127.0.0.1/bind 127.0.0.1/" /etc/redis.conf

# Step 3: 启用保护模式
sudo sed -i "s/protected-mode no/protected-mode yes/" /etc/redis.conf

# Step 4: 重启 Redis
sudo systemctl restart redis

# Step 5: 验证 Redis 连接
redis-cli -a your_redis_password ping
```

### 4.5 上传部署包

```bash
# Step 1: 创建部署目录
sudo mkdir -p /opt/talkabc/{config,logs,uploads,backup,scripts}
sudo chown -R talkabc:talkabc /opt/talkabc

# Step 2: 创建专用用户
sudo useradd -r -s /sbin/nologin talkabc

# Step 3: 上传部署包
scp talkabc talkabc@server:/opt/talkabc/
scp -r config/ talkabc@server:/opt/talkabc/

# Step 4: 设置环境变量
cat > /etc/profile.d/talkabc.sh << 'EOF'
export APP_ENV=prod
export DB_PASSWORD=your_secure_password
export REDIS_PASSWORD=your_redis_password
export JWT_SECRET=your_jwt_secret_key
EOF

# Step 5: 创建systemd服务
cat > /etc/systemd/system/talkabc.service << 'EOF'
[Unit]
Description=TalkABC API Server
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=talkabc
WorkingDirectory=/opt/talkabc
EnvironmentFile=/etc/profile.d/talkabc.sh
ExecStart=/opt/talkabc/talkabc
Restart=always
RestartSec=5
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF

# Step 6: 启动服务
sudo systemctl daemon-reload
sudo systemctl enable talkabc
sudo systemctl start talkabc
```

### 4.6 Nginx配置

```nginx
server {
    listen 80;
    server_name api.talkabc.com;

    # HTTP转HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.talkabc.com;

    # SSL证书
    ssl_certificate /etc/letsencrypt/live/api.talkabc.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.talkabc.com/privkey.pem;

    # 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 静态文件
    location /uploads/ {
        root /opt/talkabc;
        expires 30d;
    }

    # WebSocket
    location /ws {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API接口
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 六、启动服务

```bash
# 进入项目目录
cd backend

# 下载依赖（根据go.mod安装所有依赖包）
go mod tidy

# 启动服务（会自动创建数据表）
go run cmd/server/main.go

# 或编译后运行
go build -o talkabc.exe ./cmd/server
```

服务启动后会：
1. 加载配置文件（config.yaml）
2. 加载日志配置文件（config/logger.yaml）
3. 如果配置 `reset=1`，执行数据重置
4. 连接数据库（PostgreSQL，不存在则自动创建）
5. 自动创建所有数据表（根据models.go中的定义）
6. 初始化Redis连接
7. 初始化短信网关
8. 创建上传文件目录
9. 监听 8080 端口

### 11.1 重置功能说明

当 `system.reset=1` 时，启动程序会自动执行以下重置操作：

| 操作 | 说明 |
|------|------|
| 删除数据库 | 删除并重建 PostgreSQL 数据库 |
| 删除日志 | 删除 `./logs` 目录及所有日志文件 |
| 删除上传文件 | 删除 `./uploads` 目录及所有上传文件 |
| 清空Redis | 清空 Redis 当前数据库中的所有数据 |

### 11.2 使用方式

**配置文件方式：**
```yaml
system:
  reset: 1
```

**命令行方式：**
```bash
./talkabc.exe --system-reset=1
```

### 11.3 注意事项

1. **谨慎使用**：重置操作会删除所有数据，无法恢复
2. **开发环境**：建议仅在开发环境使用此功能
3. **重置后**：数据库会自动重建并执行迁移，无需手动操作
4. **自动重置**：重置完成后程序会继续正常启动
